# Contributing Guide

Thank you for considering contributing to the AI Resume Analyzer!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch for your feature
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Setup

Follow the [QUICKSTART.md](QUICKSTART.md) guide to set up your development environment.

## Code Style

### Python (Backend)
- Follow PEP 8 style guide
- Use type hints where possible
- Write docstrings for functions
- Keep functions focused and small

### TypeScript/React (Frontend)
- Use TypeScript for type safety
- Follow React best practices
- Use functional components with hooks
- Keep components small and reusable

### General
- Write clear, descriptive commit messages
- Comment complex logic
- Keep code DRY (Don't Repeat Yourself)
- Write self-documenting code

## Testing

### Backend
```bash
cd backend
python test_api.py
```

### Frontend
```bash
cd frontend
npm run lint
npm run build
```

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update FEATURES.md if adding features
5. Write clear PR description
6. Link related issues

## Feature Requests

Open an issue with:
- Clear description
- Use case
- Expected behavior
- Any relevant examples

## Bug Reports

Open an issue with:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment details

## Areas for Contribution

### High Priority
- Additional resume checks
- Improved AI prompts
- Better error handling
- Performance optimizations
- UI/UX improvements

### Medium Priority
- Additional file formats
- Export functionality
- Resume templates
- Analytics features

### Low Priority
- Internationalization
- Dark mode
- Accessibility improvements
- Additional integrations

## Questions?

Open an issue with the "question" label.
